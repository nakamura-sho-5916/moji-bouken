# プロジェクト概要

プロジェクト名：
もじぼうけん！

目的：
小学2年生を主対象とする、ひらがな・カタカナ学習RPG。
単なる文字暗記ではなく、以下の学習過程を育てる。

1. 形を知る
2. 形を探す
3. 意味と結び付ける
4. 組み合わせる
5. 表出する
6. 文章や生活で活用する
7. 他教科へ応用する

# UI原則

- 小学2年生が一人で操作できる
- 1画面1目的
- 大きな文字とボタン
- 基本操作はタップ
- 長い説明文を避ける
- 音声問題は初期版では使用しない
- 色だけで正解・不正解を表現しない
- 不正解を責める表現は禁止
- 制限時間やカウントダウンを表示しない
- 子ども画面では数値による達成率を中心にしない
- 世界の復興、仲間、装備、新エリアで成長を見せる
- 数値データは主に保護者画面で表示する

# 学習アルゴリズム

- 3回以上出題された文字を苦手判定対象とする
- 正答率70％以下を苦手文字の基準とする
- 苦手文字は翌日、3日後、7日後に復習する
- 前回より短時間で正解した場合は改善ボーナス対象とする
- 制限時間は設けない
- 苦手克服時は通常より大きな報酬と演出を付与する

# コーディング規約

- TypeScript strictを使用する
- anyを原則使用しない
- 1コンポーネント1責務
- ビジネスロジックとUIを分離する
- 学習ロジックは純粋関数としてテスト可能にする
- マジックナンバーを避ける
- 型定義を明確にする
- エラー処理を省略しない
- 未使用コードを残さない
- 既存機能を勝手に削除しない
- 大規模変更前は変更範囲を確認する

# 作業完了条件

各Issue完了時は可能な範囲で以下を実行する。

- lint
- typecheck
- unit test
- build
- 必要な場合はE2E test

作業後は以下を報告する。

1. 完了したIssue
2. 作成・変更したファイル
3. テスト結果
4. 動作確認方法
5. 未解決事項
6. 次に進めるIssue

# Gitルール

- Git実行ファイルは、PATH未反映時には以下を使用する
  C:\Program Files\Git\cmd\git.exe
- mainブランチを使用する
- ユーザーの明示指示がない限りcommitしない
- reset --hard、clean -fd、強制削除は禁止
- .gitフォルダを削除・移動しない
- .git-empty-backupは削除せず、.gitignore対象のまま保持する
